Introducción a Angular

¿Para qué sirve?

Angular es un framework de desarrollo frontend que permite crear aplicaciones web dinámicas, modernas y escalables. Está diseñado para gestionar vistas basadas en datos en tiempo real, optimizando la experiencia del usuario.

¿Qué aplicaciones tiene?

Angular se utiliza en una variedad de contextos, como:

  • Aplicaciones de una sola página (SPA, Single Page Applications).
  • Sistemas empresariales complejos como CRMs y ERPs.
  • Aplicaciones móviles híbridas mediante frameworks como Ionic.
  • Paneles de administración y sistemas de análisis de datos.

¿Sobre qué lenguajes se apoya?

Angular utiliza principalmente TypeScript, un superconjunto de JavaScript que mejora la productividad y el mantenimiento del código. También emplea:

  • HTML para las vistas y estructura del contenido.
  • CSS o SCSS para los estilos.

¿Qué prerrequisitos necesita?

Para comenzar con Angular, necesitas conocimientos básicos de:

  • JavaScript: Lenguaje base para comprender la lógica de Angular.
  • HTML: Para estructurar las vistas.
  • CSS: Para estilizar los componentes.
  • TypeScript: No es obligatorio, pero es recomendable para sacar el máximo partido a Angular.

Instalación de Angular

¿Se precisa instalación?

Sí, Angular requiere la instalación de herramientas específicas para crear y ejecutar proyectos. Esto incluye Node.js y Angular CLI, que son esenciales para desarrollar con este framework.

¿Qué pasos hay que realizar para realizar la instalación?

Sigue estos pasos para instalar Angular:

  1. Instala Node.js desde nodejs.org.
  2. Verifica que Node.js y npm están instalados ejecutando node -v y npm -v en una terminal.
  3. Instala Angular CLI globalmente con el siguiente comando: 
    npm install -g @/cli
  4. Verifica la instalación de Angular CLI ejecutando: 
    ng version

¿Versiones necesarias?

Las versiones recomendadas son:

  • Node.js: Versión 14.x o superior (preferiblemente LTS).
  • Angular CLI: Versión más reciente, obtenida al instalar mediante npm.

¿Rutas para tener en cuenta?

Angular CLI instala herramientas globalmente en el sistema. Asegúrate de que la ruta de Node.js y npm esté configurada correctamente en tu entorno: 

echo $PATH

¿Variables de entorno?

Por lo general, no necesitas configurar variables de entorno adicionales para trabajar con Angular. Sin embargo, verifica que Node.js y npm estén en el PATH de tu sistema para que puedas ejecutar comandos desde cualquier directorio.

¿Necesita base de datos?

Angular no requiere una base de datos para funcionar, ya que es un framework frontend. Sin embargo, puede consumir APIs que interactúan con bases de datos. Por ejemplo, podrías usar servicios RESTful o GraphQL para conectar con una base de datos externa.

¿Hay herramientas opcionales útiles?

Sí, algunas herramientas pueden facilitar el desarrollo:

  • Visual Studio Code: Un editor de código ligero con soporte para Angular y TypeScript.
  • Extensiones para VS Code: Angular Language Service para autocompletado y soporte de plantillas.
  • Postman: Para probar y depurar APIs que tu aplicación Angular consuma.

Primeros Pasos con Angular

¿Cómo realizar una primera aproximación ("Hola Mundo")?

A continuación, se explican los pasos para crear un proyecto Angular y mostrar un mensaje "Hola Mundo" en la pantalla.

1. Crear un nuevo proyecto Angular

Abre una terminal y ejecuta el siguiente comando para crear un nuevo proyecto Angular: 

ng new hola-mundo

Angular CLI te pedirá algunas opciones sobre el proyecto. Puedes aceptar las predeterminadas presionando Enter.

2. Navegar al directorio del proyecto

Una vez creado el proyecto, navega al directorio del proyecto con el siguiente comando: 

cd hola-mundo

3. Ejecutar el servidor de desarrollo

Inicia el servidor de desarrollo para visualizar la aplicación ejecutando: 

ng serve --open

Esto abrirá tu navegador en la dirección http://localhost:4200, donde verás la página predeterminada de Angular.

4. Modificar el componente principal

Abre el archivo src/app/app.component.html en tu editor de código. Este archivo controla la vista principal de la aplicación. Reemplaza su contenido con el siguiente código: 

<h1>¡Hola Mundo desde Angular!</h1>

5. Guardar y visualizar

Guarda los cambios y actualiza el navegador (si no se actualiza automáticamente). Verás el mensaje "¡Hola Mundo desde Angular!" en pantalla.

¡Enhorabuena!

Has creado tu primer proyecto Angular y mostrado un mensaje en pantalla. Ahora estás listo para explorar más funciones de Angular.

Utilización del Framework Angular

¿Cómo empezar a crear una aplicación base?

Para comenzar a crear una aplicación base en Angular, sigue estos pasos:

  1. Abre una terminal y ejecuta el siguiente comando para crear un nuevo proyecto: 
    ng new mi-aplicacion
    Este comando genera un espacio de trabajo con una estructura inicial.
  2. Responde a las preguntas que se muestran en la terminal:
    • ¿Quieres agregar enrutamiento? (Responde "Sí" si necesitas navegación entre páginas).
    • ¿Qué formato de estilos deseas usar? (Elige entre CSS, SCSS, o LESS).
  3. Navega al directorio del proyecto con: 
    cd mi-aplicacion
  4. Inicia el servidor de desarrollo ejecutando: 
    ng serve

¿Cómo visualizar los resultados de la ejecución?

Una vez que ejecutas ng serve, el servidor local iniciará en el puerto predeterminado. Para visualizar los resultados:

  1. Abre tu navegador web y navega a: http://localhost:4200.
  2. Verás una página de bienvenida generada automáticamente por Angular. Esto confirma que tu aplicación se ejecuta correctamente.
  3. Para personalizar la página, edita el archivo: 
    src/app/app.component.html
    Por ejemplo, reemplaza su contenido con: 
    <h1>¡Hola Mundo desde Mi Aplicación Angular!</h1>
  4. Guarda los cambios. Si el servidor está corriendo, los resultados se actualizarán automáticamente en el navegador gracias a la recarga en vivo.

Explicación del Funcionamiento del Framework

¿Actúa sobre páginas, ficheros, secciones…?

Angular organiza el contenido en una estructura jerárquica basada en:

  • Componentes: Controlan una sección de la interfaz de usuario (UI).
  • Módulos: Agrupan componentes relacionados.
  • Servicios: Gestionan lógica y datos compartidos entre componentes.

Angular permite construir aplicaciones de una sola página (SPA), donde el contenido cambia dinámicamente sin necesidad de recargar toda la página.

¿Cómo se editan/modifican los ficheros que actúan en el framework?

Cada componente de Angular tiene tres archivos principales:

  • HTML: Define la estructura de la interfaz (por ejemplo, botones, tablas, formularios).
  • CSS: Agrega estilos y diseño visual a la interfaz.
  • TypeScript: Contiene la lógica y funcionalidad del componente.

Para modificar un componente, simplemente edita estos archivos en tu editor de texto o IDE, como Visual Studio Code. Angular recargará automáticamente los cambios si el servidor de desarrollo está corriendo.

¿Cómo se referencian los ficheros creados desde otras aplicaciones (si es el caso)?

Para referenciar ficheros creados en Angular desde otras aplicaciones o proyectos:

  1. Exporta el componente o módulo: Asegúrate de que el componente esté exportado desde su módulo correspondiente.
  2. Compártelo como una librería: Angular permite empaquetar componentes en librerías reutilizables que puedes publicar en un repositorio como NPM.
  3. Usa el selector del componente: En un proyecto Angular, los componentes pueden ser referenciados directamente en el HTML mediante su selector (por ejemplo, <app-my-component></app-my-component>).

Si necesitas integrar Angular con aplicaciones externas (no Angular), puedes exportar los elementos como un paquete JavaScript o incrustar las vistas Angular dentro de un iframe.
